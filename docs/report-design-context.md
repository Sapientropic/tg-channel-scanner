# Report Design Context

## Users

- Primary audience: People using Telegram channel scans as a signal report for jobs, market monitoring, airdrops, news, or other curated watchlists.
- Usage context: Generated single-file HTML reports are read locally, shared as screenshots, and shown in README/demo materials.
- Job to be done: Scan the report quickly, understand the strongest matches, inspect why they were selected, and open source context when needed.

## Brand And Tone

- Brand personality: pixel, precise, cultured.
- Emotional target: A polished retro signal deck that feels readable for daily decisions and distinctive enough for public screenshots.
- Must not feel like: dirty brown-green paper, neon cyberpunk, toy-like pixel art, generic dashboard card mosaic, AI-gradient landing page, or decorative chrome that slows scanning.

## Visual Direction

- Preferred aesthetic: Premium retro pixel intelligence report, with a strong masthead, dashboard-like stats, pixel icons, restrained semantic color, and careful text rhythm.
- Preferred density: Medium-high information density on desktop, compressed but readable on mobile.
- Color constraints: Tinted paper/ink light mode and night reading dark mode; OKLCH tokens; no pure black or white.
- Typography constraints: Readable technical display type for titles, highly readable sans for body text, pixel styling reserved for icons, borders, and meters; robust CJK fallbacks.
- Motion constraints: Purposeful entrance, theme transition, raw-source reveal, subtle scan/tick motion, and scroll parallax on background/frame layers only; text must stay flat and readable; respect reduced motion.

## Product Constraints

- Existing assets: Embedded product icon plus README wordmark; current report semantics use high, medium, and low priority colors.
- Preserve: Single-file HTML output, report stats, profile facts, priority sections, result cards, raw Telegram source expansion.
- Viewports: Desktop screenshots for README/demo, mobile HTML reading, and browser-safe local preview.

## Principles

- Lead with the report as a premium retro pixel intelligence deck, not a generic admin dashboard.
- Keep semantic color rare and legible so priority remains a decision tool.
- Preserve source traceability and raw-message reading as first-class report behavior.
